# مسیریابی دی‌ان‌اس هوشمند (dns-smart-routing) برای OpenWRT

پکیج سبک حل مشکل قطعی و اختلال دی‌ان‌اس در روترهای OpenWRT.

این ابزار به کاربران OpenWRT که در شبکه‌های دارای اختلال شدید یا فیلترینگ (مانند ایران) با قطعی و عدم دسترسی به وب‌سایت‌ها مواجه هستند کمک می‌کند.

## این ابزار چه مشکلی را حل می‌کند؟
در شبکه‌های دارای اختلال شدید، درخواست‌های دی‌ان‌اس عمومی (WAN DNS) معمولاً با مسموم‌سازی دی‌ان‌اس (DNS Poisoning)، تایم‌اوت و کندی فراوان مواجه می‌شوند. این برنامه به صورت مداوم پایداری اتصال دی‌ان‌اس را بررسی کرده و در صورت تشخیص قطعی، ترافیک درخواست‌های دی‌ان‌اس روتر را به یک حل‌کننده امن محلی سوییچ می‌کند تا از قطع شدن اتصال کاربران جلوگیری شود.

---

## چه کارهایی انجام می‌دهد:
- **پایش سلامت دی‌ان‌اس**: پایش وضعیت اتصال و حل دی‌ان‌اس با ارسال درخواست به سرورهای `1.1.1.1` و `8.8.8.8`.
- **سوییچ هوشمند و بدون قطعی**: سرورهای دی‌ان‌اس dnsmasq را بدون اختلال یا قطع شدن ارتباط فیزیکی بازنشانی و سوییچ می‌کند.
- **حفظ وضعیت بهینه**: استفاده از یک میکرومدل دو حالته (NORMAL و FAILOVER) پایدار و سبک.

## چه کارهایی انجام نمی‌دهد (سلب مسئولیت مهم):
- ❌ **فیلترشکن یا پروکسی نیست**: ترافیک شبکه شما را پروکسی یا تونل نمی‌کند.
- ❌ **به تنهایی فیلترینگ را دور نمی‌زند**: جایگزین هسته‌هایی مثل Passwall یا Xray نیست، بلکه به عنوان مکمل پایداری در کنار آن‌ها کار می‌کند.
- ❌ **تغییری در فایروال یا مسیریابی ایجاد نمی‌کند**: هیچ تغییری در جداول روت یا قوانین فایروال روتر (iptables/nftables) ثبت نمی‌کند.

---

## نحوه کارکرد (مدل دو وضعیتی)
- **NORMAL**: دی‌ان‌اس WAN پایدار است. فایل دی‌ان‌اس‌های پویا خالی است و درخواست‌ها از طریق سرورهای دی‌ان‌اس پیش‌فرض اینترنت حل می‌شوند.
- **FAILOVER**: در صورت بروز **۲ خطای متوالی** در پایش سرورهای عمومی، سیستم به وضعیت FAILOVER سوییچ شده و تمامی درخواست‌ها به سمت دی‌ان‌اس محلی امن هدایت می‌شوند.
- **بازگشت به حالت عادی**: بلافاصله پس از ثبت **۱ موفقیت** در پایش دی‌ان‌اس، سیستم مجدداً به وضعیت NORMAL باز می‌گردد.

## نحوه نصب

### روش اول: استفاده از اسکریپت نصب خودکار (توصیه شده)
```bash
wget -O - https://raw.githubusercontent.com/Recoba86/openwrt-dns-smart-routing/main/install.sh | sh
```

### روش دوم: نصب دستی فایل IPK
فایل `.ipk` را از [صفحه ریلیز نسخه v1.0.0](https://github.com/Recoba86/openwrt-dns-smart-routing/releases/tag/v1.0.0) دانلود کرده و دستورات زیر را اجرا کنید:
```bash
opkg update
opkg install jq
opkg install /tmp/dns-smart-routing_1.0.0.ipk
/etc/init.d/dns-smart-routing enable
/etc/init.d/dns-smart-routing start
```

## تنظیمات و پیکربندی
این پکیج از رابط استاندارد UCI در OpenWRT استفاده می‌کند. فایل پیکربندی در مسیر `/etc/config/dns-smart-routing` قرار دارد:

```ini
config dns-smart-routing 'global'
    option enabled '1'
    option local_dns '127.0.0.1#15353'
```
- **enabled**: مقدار `1` برای فعال‌سازی و `0` برای غیرفعال کردن پایشگر پس‌زمینه.
- **local_dns**: آدرس هدف دی‌ان‌اس محلی شما (به طور پیش‌فرض `127.0.0.1#15353` که پورت دی‌ان‌اس پیش‌فرض Xray/Passwall است).

## نحوه استفاده و بررسی وضعیت

### بررسی وضعیت فعلی سیستم:
```bash
cat /etc/dns-smart-routing/state.json
```
*(خروجی به صورت `{"state":"NORMAL"}` یا `{"state":"FAILOVER"}` خواهد بود)*

### بررسی فایل سرورهای فعال dnsmasq:
```bash
cat /tmp/dnsmasq_dynamic_servers.conf
```

### بررسی وضعیت سرویس سیستم:
```bash
/etc/init.d/dns-smart-routing status
```

## نیازمندی‌ها و سازگاری
- **نسخه‌های OpenWRT**: سازگار با نسخه‌های 19.07, 21.02, 22.03, 23.05 و جدیدتر.
- **حل‌کننده دی‌ان‌اس**: به صورت بومی با سرویس **dnsmasq** کار می‌کند.
- **وابستگی‌ها**: صرفاً به ابزار `jq` نیاز دارد. بدون نیاز به نصب netcat/nc.

## امنیت و پایداری
- بدون نیاز به فایروال یا قوانین روتینگ.
- نوشتن کاملاً اتمیک وضعیت جهت جلوگیری از خرابی فایل‌سیستم در قطعی‌های ناگهانی برق روتر.
- تفکیک کامل پایشگر و اعمال‌کننده وضعیت جهت جلوگیری از تداخل یا بن‌بست پردازش‌ها.
